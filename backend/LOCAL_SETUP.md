# Local Development Setup

Complete guide for setting up the Job Tracker backend locally with PostgreSQL.

## Prerequisites

- Python 3.9+
- PostgreSQL 16+ (via Homebrew)
- macOS/Linux (Windows users: use WSL2)

## Quick Start (5 minutes)

### 1. Install PostgreSQL

```bash
brew install postgresql@16
brew services start postgresql@16
```

### 2. Create Database

```bash
createdb jobtracker
```

### 3. Set Up Python Environment

```bash
cd backend
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 4. Generate Secrets

```bash
python setup.py
```

This creates `.env` with secure JWT and encryption keys.

### 5. Configure Database

The `.env` file should have:

```bash
DATABASE_URL="postgresql://YOUR_USERNAME@localhost/jobtracker"
```

Replace `YOUR_USERNAME` with your system username (run `whoami` to check).

### 6. Start Server

```bash
uvicorn app.main:app --reload
```

Server runs at: http://localhost:8000

### 7. Test It

```bash
curl http://localhost:8000/health
# Should return: {"status":"healthy","environment":"development","database":"connected"}
```

## Troubleshooting

### "Could not connect to server"

PostgreSQL not running:

```bash
brew services restart postgresql@16
```

### "Database does not exist"

```bash
createdb jobtracker
```

### "Permission denied"

Check your username:

```bash
whoami  # Use this in DATABASE_URL
```

### "Module not found"

Activate venv and reinstall:

```bash
source venv/bin/activate
pip install -r requirements.txt
```

## Environment Variables

`.env` file structure:

```bash
# Database
DATABASE_URL="postgresql://username@localhost/jobtracker"

# Security (auto-generated by setup.py)
JWT_SECRET_KEY=<64-char-hex>
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
ENCRYPTION_KEY=<base64-key>

# Application
ENVIRONMENT=development
DEBUG=True

# CORS (for frontend)
CORS_ORIGINS=http://localhost:5173,http://localhost:3000

# External APIs (optional)
GREENHOUSE_API_KEY=
WORKDAY_CLIENT_ID=
WORKDAY_CLIENT_SECRET=
```

## Development Workflow

### Making Changes

1. Edit code
2. Server auto-reloads (thanks to `--reload`)
3. Test in browser/curl
4. Check logs in terminal

### Database Migrations

When you change models:

```bash
# Create migration
alembic revision --autogenerate -m "Description"

# Apply migration
alembic upgrade head
```

### Resetting Database

```bash
# Drop and recreate
dropdb jobtracker
createdb jobtracker

# Restart server (tables auto-create)
uvicorn app.main:app --reload
```

## Security Best Practices

✅ **Already Implemented:**
- `.env` in `.gitignore` (never commit secrets!)
- Password hashing (PBKDF2)
- JWT tokens with expiration
- CORS configuration
- Security headers (CSP, X-Frame-Options, etc.)

⚠️ **For Production:**
- Use strong `JWT_SECRET_KEY` (auto-generated ✓)
- Enable HTTPS
- Set `DEBUG=False`
- Use connection pooling
- Set up proper logging
- Configure rate limiting
